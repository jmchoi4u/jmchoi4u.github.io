블로그 사용 설명서

이 문서는 컴퓨터 초보자 기준으로 만든 쉬운 설명서입니다. 블로그 글 쓰기, 연락처 바꾸기, 프로필 사진 바꾸기, 로컬 미리보기, GitHub Pages 배포까지 한 번에 따라할 수 있게 정리했습니다.

글은 어디에 쓰나?
_posts 폴더에 새 마크다운 파일을 만들면 됩니다.
프로필 사진은 어디서 바꾸나?
assets/img/profile/avatar.png 파일과 _config.yml의 avatar: 값을 확인합니다.
연락처는 어디서 바꾸나?
_config.yml과 _data/contact.yml을 수정합니다.
미리보기는 어떻게 하나?
터미널에서 bundle exec jekyll serve --livereload --open-url를 실행합니다.

1. 가장 쉬운 작업 순서

  1. 옵시디언에서 먼저 초안을 정리합니다.
  2. app/start-editor.bat 를 실행해 블로그 작성 도우미 웹앱을 엽니다.
  3. 웹앱의 본문 칸에 글을 붙여넣고, 제목/카테고리/태그를 고릅니다.
  4. 이미지는 웹앱에서 저장한 뒤 생성된 마크다운 문법을 본문에 넣습니다.
  5. 웹앱에서 미리보기 또는 정적 빌드 테스트를 합니다.
  6. 웹앱의 GitHub 발행 버튼으로 올리면 GitHub Actions가 자동 배포합니다.
가장 단순한 방식은 글 1개를 수정한 뒤 바로 로컬 미리보기로 확인하고, 이상 없으면 GitHub에 올리는 흐름입니다.

2. 프로필 사진 바꾸기

파일 위치

현재 프로필 사진 파일은 assets/img/profile/avatar.png 입니다. 사이트 설정 파일 _config.yml 에는 아래처럼 연결되어 있습니다. 

avatar: /assets/img/profile/avatar.png

왜 적용이 안 된 것처럼 보일 수 있나?

  • Chirpy 프로필 사진은 매우 작은 원형 이미지로 표시됩니다. 전신 일러스트를 넣으면 얼굴이 작아 보여서 안 바뀐 것처럼 느껴질 수 있습니다.
  • 브라우저 캐시 또는 PWA 캐시 때문에 이전 사진이 남아 있을 수 있습니다.
  • 온라인 사이트는 아직 GitHub 배포가 끝나지 않았을 수 있습니다.

가장 좋은 사진 규칙

  • 정사각형 이미지가 가장 좋습니다. 추천 크기: 800x800
  • 얼굴 중심으로 잘라서 넣으면 원형 프로필에서 훨씬 잘 보입니다.
  • 파일 이름은 되도록 영어로 쓰세요. 예: avatar.png

확인 방법

  1. 새 사진을 같은 이름으로 덮어씁니다. 예: assets/img/profile/avatar.png
  2. 터미널에서 bundle exec jekyll b 또는 bundle exec jekyll serve --livereload --open-url를 실행합니다.
  3. 브라우저에서 Ctrl + F5로 강력 새로고침을 합니다.
  4. 그래도 같아 보이면 시크릿 창에서 사이트를 다시 엽니다.
현재처럼 세로형 캐릭터 그림은 실제로는 적용되어도 원형 아바타에서 얼굴이 아주 작게 나옵니다. 눈에 잘 보이게 하려면 얼굴 중심 정사각형 버전을 따로 만드는 것을 추천합니다.

4. 블로그 글은 어디서 쓰나?

추천 흐름은 옵시디언에서 초안 작성, 블로그 작성 도우미 웹앱에서 발행용 정리 입니다. 자세한 연동 방법은 obsidian-guide.html 문서를 같이 보시면 됩니다.

글 파일을 만드는 폴더

새 글은 _posts 폴더에 만듭니다. 파일 이름 규칙은 꼭 지켜야 합니다. 

YYYY-MM-DD-파일이름.md

예시

2026-03-13-first-post.md
2026-03-13-jekyll-setting-note.md
2026-03-14-fix-github-pages-build.md

중요 규칙

  • 날짜는 파일 이름 맨 앞에 넣습니다.
  • 확장자는 .md 입니다.
  • 파일 이름은 되도록 영어, 숫자, 하이픈으로만 만드세요.
  • 글 내용은 한국어로 써도 괜찮습니다.
가장 안정적인 방법은 파일 이름은 영어로, 실제 글 내용은 한국어로 작성하는 방식입니다.

8. 로컬 테스트 방법

웹앱으로 하는 가장 쉬운 방법

  1. app/start-editor.bat 실행
  2. 웹앱에서 미리보기 시작 버튼 클릭
  3. 또는 정적 빌드 테스트 버튼 클릭

가장 많이 쓰는 명령어

bundle install
bundle exec jekyll serve --livereload --open-url

설명

  • bundle install: 처음 한 번 또는 라이브러리 변경 후 설치
  • bundle exec jekyll serve --livereload --open-url: 사이트를 로컬에서 띄우고 자동 새로고침
  • bundle exec jekyll b: 서버 없이 정적 빌드만 확인

주소

브라우저에서 보통 http://127.0.0.1:4000 으로 열립니다.

9. GitHub Pages 배포 방법

  1. 파일을 저장합니다.
  2. GitHub에 커밋하고 푸시합니다.
  3. GitHub 저장소의 Actions 탭에서 배포 진행 상태를 확인합니다.
  4. 완료되면 https://jmchoi4u.github.io 에 반영됩니다.
이 저장소에는 이미 GitHub Actions 배포 워크플로가 들어 있습니다. 그래서 보통은 푸시만 하면 됩니다.

10. SEO 기본 설명

지금 구조도 검색 노출은 가능합니다. Jekyll과 Chirpy는 기본 메타 태그, 사이트맵, 글 구조를 잘 만들어주는 편입니다.

한국어 블로그인데 영어권 사람도 들어오게 할 수 있나?

가능합니다. 다만 방법이 중요합니다.

  • 한국어 글을 주로 쓰되, 중요한 기술 용어는 영어 원문도 같이 적습니다.
  • 글 맨 위나 맨 아래에 짧은 English Summary를 넣으면 영어 검색 사용자에게 도움이 됩니다.
  • 정말 영어 유입을 늘리고 싶다면 영어 전용 글을 별도 URL로 만드는 것이 가장 좋습니다.
  • 제목, 설명, 본문이 모두 한국어인데 영어 검색까지 크게 기대하는 것은 어렵습니다.
한국어 중심 글 한 페이지 안에 한국어와 영어를 무리하게 반반 섞으면 검색엔진이 언어를 애매하게 이해할 수 있습니다. 가장 좋은 방식은 한국어 본문 + 짧은 영어 요약, 또는 한국어 글과 영어 글을 각각 따로 작성하는 것입니다.

11. 자주 생기는 문제

프로필 사진이 안 바뀌어 보일 때

  • 캐시일 수 있으니 Ctrl + F5를 눌러보세요.
  • 온라인 사이트는 GitHub Actions 배포가 끝났는지 확인하세요.
  • 사진이 너무 복잡하면 이미지는 바뀌었는데 티가 안 날 수 있습니다.

글이 목록에 안 보일 때

  • 파일 이름이 YYYY-MM-DD-이름.md 규칙인지 확인하세요.
  • 맨 위의 --- 앞뒤 설정이 올바른지 확인하세요.
  • 날짜가 미래 날짜인지 확인하세요.

로컬 서버가 안 뜰 때

  • bundle install을 먼저 실행하세요.
  • 터미널에 나온 에러 문장을 그대로 복사해 확인하세요.
  • Windows에서는 가끔 브라우저 캐시보다 새 터미널로 다시 실행하는 것이 빠릅니다.
