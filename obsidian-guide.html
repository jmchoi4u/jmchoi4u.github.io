옵시디언 연동 설명서

가장 현실적인 사용 흐름은 옵시디언에서 초안 작성 후 블로그 작성 도우미 웹앱으로 옮겨 저장하는 방식입니다. 이 방법이 글 정리와 발행을 가장 깔끔하게 나눌 수 있습니다.

1. 추천 작업 순서

  1. 옵시디언에서 글 초안을 자유롭게 정리합니다.
  2. 제목, 본문, 코드 블록, 체크리스트를 먼저 다듬습니다.
  3. 이미지는 임시로 붙여넣지 말고 따로 보관합니다.
  4. 정리된 본문을 블로그 작성 도우미 웹앱에 붙여넣습니다.
  5. 웹앱에서 날짜, 카테고리, 태그, true/false 옵션을 고릅니다.
  6. 이미지를 웹앱으로 저장한 뒤 생성된 마크다운 문법을 본문 중간에 넣습니다.
  7. 미리보기 또는 정적 빌드 테스트 후 GitHub 배포 버튼을 누릅니다.

2. 옵시디언에서 어떻게 써두면 좋은가?

좋은 예

  • 제목은 첫 줄에 적어두기
  • 본문은 마크다운 그대로 작성하기
  • 코드 블록은 언어명까지 함께 적기
  • 태그 후보를 맨 아래에 메모해두기

덜 좋은 예

  • 옵시디언 전용 문법을 과하게 쓰기
  • 이미지를 바로 로컬 절대경로로 링크 걸기
  • 한글/공백/특수문자가 섞인 긴 파일명을 이미지에 사용하기

3. 이미지 파일 위치를 깔끔하게 관리하는 방법

게시글별 폴더를 따로 만드는 방식이 가장 깔끔합니다.

assets/
  img/
    posts/
      2026-03-13-first-post/
        cover.png
        screenshot-01.png
        screenshot-02.png
이미지를 옵시디언 안에 직접 끌어다 넣기보다, 웹앱의 이미지 저장 기능으로 assets/img/posts/글폴더명/ 아래에 넣는 습관이 가장 안정적입니다.

4. 웹앱으로 옮길 때 실제로 하는 일

  1. 옵시디언 본문을 전체 복사합니다.
  2. 웹앱의 본문 칸에 붙여넣습니다.
  3. 제목, slug, 카테고리, 태그를 정리합니다.
  4. 날짜는 웹앱이 자동으로 넣어줍니다.
  5. TOC, comments, pin, mermaid, math 옵션을 체크합니다.
  6. 이미지는 웹앱의 이미지 저장 영역에서 파일을 저장합니다.
  7. 생성된 ![이미지 설명](경로) 문법을 본문 원하는 위치에 넣습니다.

5. 옵시디언에서 쓸 때 추천 템플릿

# 글 제목

## 한 줄 요약

요약을 먼저 적습니다.

## 본문

내용을 적습니다.

## 코드

```bash
bundle exec jekyll serve
```

## 태그 후보

- chirpy
- github-pages
- jekyll

6. 꼭 기억할 점

  • 옵시디언은 초안 정리용, 웹앱은 블로그 발행용으로 역할을 나누면 편합니다.
  • 이미지 경로는 웹앱에서 정리하는 쪽이 훨씬 덜 꼬입니다.
  • 파일 이름과 이미지 이름은 영어/숫자 중심으로 짧게 만드는 것이 좋습니다.
  • 최종 게시 전에 웹앱에서 로컬 미리보기나 정적 빌드 테스트를 먼저 하세요.
