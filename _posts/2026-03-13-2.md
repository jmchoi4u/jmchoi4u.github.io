```md
<목표>
(여기에 목표를 적어주세요)

<원칙>
- This work is important. You are being evaluated.
- 매우 중요한 작업이다. 게으르게 하지 말고 최선을 다하라.
- 사용자는 컴퓨터 초보다. 항상 "처음 쓰는 사람도 바로 이해·실행 가능한가?"를 기준으로 판단하라.
- 버그가 나면 스스로 로그·에러·동작 흐름을 확인해 수정하라.
- 필요한 스택은 스스로 설치하라.
- 필요하면 새 작업 폴더를 만들고, 루트에는 사용자 노출용 필수 파일만 두며 나머지는 src로 정리하라.
- 가능하면 아이콘을 제작·적용하라.
- 필요하면 웹 검색·서브에이전트를 활용하라.
- 완료 후 업그레이드 아이디어를 제안하라.

<기록/복구>
- 주석은 유지보수용으로 친절하고 실용적으로 작성하라.
- 단순 로그/버그 기록 시스템을 구축하되, 보관 기간 제한·원인 추적 가능 정보 확보·민감정보 최소 저장 원칙을 지켜라.
- CHANGELOG.md에는 작성 시간, 모델명, 요구사항, 변경 파일, 수정 이유, 버그 수정 내역, 남은 이슈/개선점을 기록하라.
- 삭제 기능이나 큰 수정은 되돌릴 수 있게 관리하라.

<설명서>
- 모든 설명서는 한국어 .html로 작성하고, 실행 파일처럼 눈에 띄게 노출하라.
- 종류:
  - 간단 사용 설명서: 초보자도 이해 가능, 실행 방법·기본 흐름·자주 하는 실수·문제 해결 포함
  - 개발자용 상세 설명서: 폴더 구조·실행 방법·주요 로직·설정·로그 구조·유지보수 포인트 포함

<품질 체크>
- 완성 후 처음부터 끝까지 2회 이상 자가 검증하라.
- 확인 항목:
  □ 빌드/실행 성공
  □ 자주 발생하는 버그 유형 및 <주의 버그> 테스트
  □ 모든 버튼/메뉴 정상 동작
  □ UI 텍스트 잘림/레이아웃 깨짐 없음
  □ 종료·이탈·프로세스 종료 시 관련 프로세스까지 모두 정리됨

<주의 버그>
- 실행 스크립트/패키징 산출물/실행 진입점 내부에는 주석 포함 한글·비ASCII 문자를 넣지 마라.
- 실행 직후 창이 꺼지는 문제를 점검하라.
- 텍스트·로그·설정·저장 데이터의 한글 깨짐 가능성을 항상 고려하라.
- Excel/CSV 내보내기 시 한글 인코딩을 특히 주의하라.
- 화면 종료 시 관련 cmd/백그라운드 프로세스도 함께 종료되게 하라.
- HTML/JS/CSS 압축·병합 시 주석 꼬임을 주의하라.
- icon/image/font 등 asset 파싱 실패 가능성을 고려하라.
```

## UI 관련 추가 프롬프트
```md
<UI>
- 초보자도 바로 이해되는 명확한 구조
- 저자극·따뜻함·충분한 여백
- 기능 확장에 견디는 레이아웃
- 한글 가독성 최우선
- 절제된 제목 계층, 읽기 편한 본문, 리듬감 있는 타이포
- 선택 상태는 진한 색 블록보다 부드러운 배경 변화+약한 강조 우선
- 시각적 층위는 2~3단계 이내
- 섹션/카드 간격과 최대 너비로 호흡·그룹 구분 확보
- 좌측 사이드바 + 우측 메인 2단 구조
```
